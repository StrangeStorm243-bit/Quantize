## Minor observations — resolution (2026-07-11)

Two were real UX defects and are **fixed** (TDD, gate-green); two are **not code defects** and are kept
as documented rationale.

- **O2 — "Open" ignored the listed version. · FIXED.** Recent-strategies rows showed per-version
  entries (v1, v2) each with an Open button, but Open always loads the **latest** version (opening "v1"
  loaded v2). Fix (`web/src/components/Home.tsx`): collapse the API's per-`(id, version)` rows to one
  row per strategy at its latest version — presentation-only dedup (invariant 5), so the shown version
  matches what Open loads. Test: `Home.test.tsx` "collapses per-version rows to one row per strategy at
  its latest version" (RED→GREEN). **Verified live:** Recent now shows *ETF Momentum Rotation · v2*
  once (the duplicate v1 row is gone).
- **O4 — Dataset picker stayed open after Select. · FIXED.** Selecting a dataset updated the dialog in
  place rather than closing. Fix (`web/src/App.tsx`): the picker's `onSelectDataset` now also
  `setDatasetPickerOpen(false)` — App-local, so Home's inline `DatasetPanel` (no modal) is unaffected.
  Test: `App.datasetPicker.test.tsx` "closes the picker when a dataset is selected" (RED→GREEN).
  **Verified live:** selecting the dataset dismisses the modal.
- **O1 — Save 409 → auto-bump console line. · BY DESIGN (not fixed).** Saving a changed doc POSTs at
  the current version, gets `409 Conflict` (immutable store), and the client bumps the version + retries
  — a deliberate, tested recovery (`App.save.test.tsx` "on a 409 conflict, bumps the version and
  retries…"). The browser logs *any* 409 response at the network layer regardless of how JS handles it;
  it cannot be suppressed without replacing the reactive try→recover protocol with a predictive
  pre-check (a design change to a correct, guarded flow, for a purely cosmetic console line). Left as-is.
- **O3 — Inner component nodes aren't inspectable. · DEFERRED (needs founder sign-off).** Inside the
  M13.8 **structurally read-only** component view you can see and trace-highlight an inner node but the
  main Inspector keeps showing the component instance. Making inner nodes inspectable is a *feature*
  (a read-only, definition-scoped Inspector) spanning App selection state + Inspector node-resolution +
  Canvas click-wiring, and it touches the deliberate M13.8 read-only contract. Per scope discipline
  (CLAUDE.md), this is a founder-approved follow-on, not a closeout edit.
