under threshold; typical is 15–30× under. No tap approached 1 s.

## Anomalies / notes

- **StrictMode double-fetch (dev-mode artifact, not a product bug).** Every UI-driven tap emits
  **exactly two** `elapsed_ms` log lines; the single `curl` setup tap emitted **one**. This is React
  18 StrictMode double-invoking effects in the Vite dev server — an expected development artifact,
  not a production behavior. Recorded honestly rather than hidden. The value tap is idempotent, so
  the double fetch is harmless (identical result, no writes).
- The `cap` address shows a third log line at 21:00:07 (`elapsed_ms=68.1`) — that is **my direct
  `curl`** verifying the DTO's served `cash` field, not a third UI fire. UI `cap` taps were the two
  lines at 20:59:55.
- **React Flow console warnings (pre-existing, benign).** On graph load, React Flow logs
  `Couldn't create edge for target handle id: "series"` / `source handle id: "values"` for the
  component instance's ports (error#008). These are unrelated to the value tap and did not affect any
  render; noted for completeness. The only other console error is a missing `favicon.ico` (404).
- No empty-200 responses and no blank Inspector sections were observed at any step; every refusal
  rendered a specific, honest message.
