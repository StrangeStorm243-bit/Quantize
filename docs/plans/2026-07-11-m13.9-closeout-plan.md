**Goal:** Make the reoriented product walkable by a first-time user (journey checklist + complete
empty states), execute the two scripted closeout instruments (fresh-profile journey walkthrough,
30-second legibility test) with recorded observations, and close out M13 (README, LEARNING_LOG,
plan DoD, smoke findings F2/F3).

**Architecture:** Pure frontend presentation + docs. One new UI cluster (`web/src/journey/`): a
pure, unit-tested progress model (step latching + localStorage persistence, following the existing
`theme.ts` graceful-failure pattern) consumed by a dumb `JourneyChecklist` component rendered from
App on both Home and editor views. Step completion is *inferred from App state the app already
owns* — no new backend, no DTO/codegen change, no engine/IR change, nothing enters the document
(invariant 1), no logic beyond presentation (invariant 5).

**Tech stack:** React + TS (`web/`), vitest + RTL, localStorage. Zero new dependencies.

### Task 1: Pure journey-progress model (TDD)

**Files:** create `web/src/journey/progress.ts`, `web/src/journey/progress.test.ts`.

The model is pure functions + a storage wrapper; no React.

**Contract (write tests first, watch them fail, then implement):**

```ts
export type JourneyStepId =
  'open-demo' | 'run-backtest' | 'open-results' | 'open-trace' | 'extract-component'
export interface JourneyStep { id: JourneyStepId; label: string }           // ordered, mirrors README §4
export const JOURNEY_STEPS: JourneyStep[]                                    // exactly the 5 above
export interface JourneyState { done: JourneyStepId[]; dismissed: boolean }  // done = monotonic latch
export const EMPTY_JOURNEY: JourneyState
/** Latch: union of already-done + newly-observed; never un-ticks. Order-stable, deduped. */
export function latchSteps(state: JourneyState, observed: JourneyStepId[]): JourneyState
/** localStorage round-trip under 'quantize.journey.v1'; corrupt/missing/inaccessible →
 *  EMPTY_JOURNEY / silent no-op. */
export function loadJourney(): JourneyState
export function saveJourney(state: JourneyState): void
```

Test list (each a separate RED→GREEN cycle): latch adds a new step; latch never removes (`done`
already containing a step stays); latch dedupes; latch preserves `dismissed`; load returns
`EMPTY_JOURNEY` on missing key; load survives corrupt JSON (`'{nope'`) and wrong-shape JSON
(`'{"done":"x"}'`); save→load round-trips; save failure (mock throwing `setItem`) does not throw.

Commit: `M13.9: journey progress model (pure latch + persisted state)`.
