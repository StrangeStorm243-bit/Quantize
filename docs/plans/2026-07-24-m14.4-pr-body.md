# M14.4 — Desktop viewport fit: dock clamp+collapse, overlay collision avoidance, inspector overflow

> **For the founder to copy into the PR description.** Branch `fix/m14.4-viewport-fit` → `main`.

## Summary

A single **bounded, pre-§13 desktop viewport-fit pass** opened by founder order as **FD-9**
(superseding FD-8's L-3 deferral) because the observed layout failures — a React Flow canvas
collapsing to as little as 40px tall, fixed-size overlays colliding, and a 260px inspector with an
internal horizontal scrollbar — would have contaminated §13 external validation, which is run *on*
those viewports. §13 remains the next step immediately after this slice.

Scope is deliberately narrow: presentation-only. Viewport-aware sizing in `App.css` (dock height
clamp, overlay media queries, scoped overflow fixes, stage-strip single-row), **one** new
`dockCollapsed` boolean in `App.tsx` routed through a single `openDock(tab)` helper, a collapse
chevron in `Dock.tsx`, and a self-owned Playwright viewport regression harness in `web/e2e/`
(opt-in; never wired into the gate or CI). No product architecture, backend semantics, DTOs,
generated `.d.ts`, `Canvas.tsx` value-tap logic, or M14.3 behavior changed. `ui.*` round-trip is
unaffected.

Target viewports: **1280×720, 1366×768, 1440×900, 1920×1080**, both themes.

## Notes for the reviewer

- **Vitest e2e-exclude incident (fixed).** Adding the Playwright spec under `web/e2e/` briefly broke
  the canonical `npm run test`: vitest collected the Playwright spec and failed. Fixed by adding
  `e2e/**` to vitest's `test.exclude` in `web/vite.config.ts` (commit `dd61ca3`). The harness now
  runs *only* via `npm --prefix web run e2e:viewport`; the unit run never sees it.
- **Harness hermeticity.** `web/e2e/run.mjs` owns a scratch temp DB and passes it via
  `QUANTIZE_E2E_DB`; the Playwright config only *reads* that var and throws loudly if unset. Two
  `webServer` entries (uvicorn:8123 with the scratch DB, Vite:5199 `--strictPort`) each carry
  explicit `cwd`/readiness/`reuseExistingServer:false`. The Vite `/v1` proxy target is
  env-overridable (`QUANTIZE_E2E_API_PROXY`, default `http://127.0.0.1:8000` — dev unchanged), set
  only inside the harness's Vite env, so the browser provably hits the scratch backend. A `setup`
  project is the single owner of run creation; the `viewport` project depends on it and carries no
  run IDs or machine-local paths.
- **Known out of scope (seams preserved).** The React Flow minimap renders light-in-dark theming —
  this is **pre-existing** (not introduced here) and out of scope for a layout-fit pass. Also
  deferred by design: extraction-toolbar fold-in, side-panel collapse, and chrome-over-*graph*
  overlap (inherent to a pannable infinite canvas; only chrome↔chrome collision was in scope).
