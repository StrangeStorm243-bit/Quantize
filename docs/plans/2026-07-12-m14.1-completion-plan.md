## Task 1 — M14.1a′: envelope-level trace cross-check

**What it is (design §6, mitigation 3):** after the recompute, compare the tapped node's
FRESH trace events against the PERSISTED events at the same
`(evaluation_instant, component_path, node_id)`. Any divergence means the current node code
no longer reproduces what the run recorded → refuse with **engine drift** rather than serve a
number the bot never acted on. Envelope-level only — an ordered comparison of
`(event_type, payload)` pairs; NO per-event-type semantic knowledge (that would be a parallel
interpretation layer — stop condition).

**Files:**
- Modify: `quantize/valuetap/service.py`
- Test: `tests/test_valuetap_crosscheck.py` (new)

**Spec:**
- New stable code in service.py: `ENGINE_DRIFT = "engine_drift"` (exported via
  `quantize/valuetap/__init__.py` alongside the existing codes).
- In `resolve_node_value`, after the value projection succeeds: load the persisted events via
  the existing `RunRepository.load_trace(run_id, session_date)` (the repo verifies stream
  hash/contiguity itself — reuse, never re-query trace_events directly), filter to
  `event.node_id == node_id and tuple(event.component_path) == component_path` (this excludes
  `engine.*` events automatically — they carry engine identity, not this node's).
- Fresh side: the same filter over `outcome.trace` (already computed for `fresh_trace`).
- **Skip silently when the persisted filtered set is empty** (a trace-less run must not make
  the tap unusable). Otherwise compare the two sequences as ordered lists of
  `(event_type, canonical_json_bytes(payload))` — use
  `quantize.persistence.serialize.canonical_json_bytes` for normalization; timestamps are
  equal by construction (both stamped with the evaluation instant) and are NOT compared.
  Persisted non-empty + fresh empty ⇒ mismatch. Any mismatch ⇒
  `ValueTapError(ENGINE_DRIFT, "engine drift — the current node implementation no longer "
  "reproduces this run's recorded trace; re-run to refresh", subject=node_id)`.
- Ordering note for the implementer: run this check BEFORE the elapsed-ms log/return, and
  only on the success path (refusals already refuse).

**⚠ CRITICAL TEST-CONSTRUCTION CORRECTION** (supersedes the M14.1a′ slice text in
`2026-07-11-m14-implementation-plan.md`, which says to tamper `trace_events` via direct DB
UPDATE): **that construction cannot work.** `RunRepository.load_trace` verifies the whole
stream against the run row's `trace_content_hash` (+ count + seq contiguity), so a tampered
row raises `PersistenceError(corrupt_artifact)` before the cross-check ever sees it. Simulate
drift HONESTLY instead: **mutate one event's payload in `result.trace` BEFORE
`RunRepository.save_run`** (trace payloads are plain mutable dicts on the in-memory
`BacktestResult`; save_run hashes whatever it is given, so the stored stream is internally
consistent yet differs from any faithful recompute — exactly what real code drift produces).
Pick a `transform.computed` event of the tapped node and alter its `computed` list.

**Tests (RED → GREEN each; reuse the module-scoped `strategy_a_result` + `_persist` fixtures
already in `tests/test_valuetap_service.py` — import or mirror minimally):**
1. Clean run → tap serves (cross-check passes silently); assert value still equals persisted
   `target_weights` (guards against the check mangling the success path).
2. Drifted run (payload mutated pre-save, per the correction above) → `ValueTapError` with
   `code == ENGINE_DRIFT` when tapping THAT node; RED first by writing the test before the
   check exists (it will serve instead of refuse).
3. Run persisted with NO trace events for the tapped node → check skipped, value serves.
   Construction: `run_backtest(..., collect_trace=False)` persists an empty stream — verify
   `save_run` accepts that (it does: zero event rows, hash over empty).
4. A *different* node's drift does not block this node's tap (mutate node X's event, tap
   node Y → serves). Proves the filter scope.
5. `engine.*` events at the instant never participate (implicitly covered by 1 on a fill
   session; make it explicit by asserting a tap succeeds at an evaluation whose instant also
   has engine events in the persisted stream).

**Steps:** write tests (expect FAIL: `ENGINE_DRIFT` unimportable / drift served) → implement →
GREEN → `./.venv/Scripts/python.exe -m pytest tests/test_valuetap_crosscheck.py tests/test_valuetap_service.py -q`
→ `ruff check` / `ruff format --check` on touched files → full
`./.venv/Scripts/python.exe -m mypy` (the orchestrator runs full mypy — focused mypy missed a
cross-file error in the last wave) → commit `feat(m14.1a'): envelope-level trace cross-check`.

## Task 2 — M14.1c: `GET /v1/runs/{run_id}/values`

**Files:**
- Modify: `quantize/api/routes/runs.py` (the M13.6 trace-tree route is the shape to copy),
  `tests/api/test_hardening.py` (**the endpoint-inventory guard at
  `test_exposed_endpoints_match_the_contract` pins the exact path set — add
  `"/v1/runs/{run_id}/values"` or the suite fails; this bit M13.6 too**).
- Test: `tests/api/test_values_endpoint.py` (new).

**Route spec:**
```python
@router.get("/{run_id}/values")
def fetch_node_value(run_id: str, node_id: str, session_date: date, settings: SettingsDep,
                     component_path: str | None = None, output_port: str | None = None
                     ) -> NodeValueResponse: ...
```
- `component_path` comma-split into a tuple; empty/None → `()`. **Re-validate every segment
  against `^[A-Za-z0-9_]+$`** (import the pattern source from `quantize/schema/primitives.py`
  rather than restringing if exportable; else define the same regex with a comment naming its
  origin) → violation raises `ApiRequestError(422, "invalid_value_address", ...)` (one code
  for the whole address — it guards node_id and output_port too, not just component_path).
  `node_id` and a present `output_port` get the same segment validation.
- Malformed `session_date` → FastAPI's own 422 (query parsing) — assert, don't reimplement.
- Body: `with Database(settings.db_path, busy_timeout_ms=settings.busy_timeout_ms) as db:`
  (per-request handler-owned pattern) → `resolve_node_value(db, ...)` →
  `summarize(resolved.value)` → `node_value_dto(...)` with
  `run_id=resolved.run_id, dataset_fingerprint=resolved.dataset_fingerprint,
  captured=resolved.captured` (never literals).
- **Error mapping** — catch `ValueTapError` and raise `ApiRequestError(status, err.code,
  err.message)` (only code+message cross the wire; `subject`/`diagnostics` stay server-side):
  `value_address_not_found` → 404 · `no_evaluation_at_session` → 404 ·
  `ambiguous_output_port` → 422 · `unknown_provenance` / `dataset_mismatch` /
  `calendar_mismatch` / `recompute_failed` / `engine_drift` → 409.
  `PersistenceError` propagates untouched (unknown run → global handler → 404
  `artifact_not_found`, exactly like the trace endpoints).

**Tests (mirror the service matrix through HTTP; use the `tests/api/conftest.py` app/db
fixtures and seed via repositories the way the existing api tests do):**
1. Happy path: seed Strategy A run + dataset, GET the `cap` node at an evaluated session →
   200; body's `asset_values` equal the persisted `target_weights`;
   `value_summary.kind == "portfolio_targets"`; `provenance.captured is False`.
2. Nested: componentized run, `component_path=<instance-id>` + `node_id=ret` → 200 with
   `kind == "cross_section"`.
3. **Every failure mode's exact status + stable `code` string**: unknown run → 404
   `artifact_not_found` · unknown node → 404 `value_address_not_found` · no-eval session →
   404 `no_evaluation_at_session` (message quotes the recorded note when present) · omitted
   port on a multi-output instance → 422 `ambiguous_output_port` · deleted dataset row → 409
   `dataset_mismatch` · unknown-provenance run (direct record swap as in the service tests) →
   409 `unknown_provenance` · drifted run (Task 1's pre-save mutation) → 409 `engine_drift`.
4. Malformed `component_path` segment (`"a,b c"` or `"a,,b"`) → 422; malformed
   `session_date` → 422.
5. **Contract test:** the 200 body validates against the committed API schema bundle via the
   `api_schema`/`def_validator` fixtures in `tests/api/conftest.py` (validate against the
   `NodeValueResponse` def — the repo's established pattern).
6. **Byte determinism through HTTP:** two identical GETs → identical `response.content`.
7. Response headers/serialization: default FastAPI JSON of the pydantic model is fine (this
   DTO needs no verbatim-bytes embedding — nothing here re-encodes a stored artifact).

**Steps:** add the route path to `test_hardening.py`'s inventory FIRST and watch the suite
fail (RED proves the guard works) → write the endpoint tests (RED: 404 route-not-found) →
implement the route → GREEN → focused pytest (`tests/api/test_values_endpoint.py tests/api/test_hardening.py`)
→ full `pytest` → ruff/format → full mypy → commit `feat(m14.1c): GET /v1/runs/{run_id}/values endpoint`.

**Explicitly NOT in Task 2:** no DTO/schema change (the DTO landed in 1b — if the route
seems to need a shape change, STOP: that is contract drift), no frontend, no OpenAPI export,
no caching, no batch/multi-port endpoint.
