## 1. Root-cause diagnosis (measured live, demo data, all four viewports)

Evidence screenshots: `.playwright-mcp/m144-01…05*.png` (2026-07-24 session).

- **RC-1 — No viewport-aware element in the vertical budget.** Fixed stacked bands: header 42px +
  strategy bar 39px + extraction toolbar 39px + stage strip 48px (78px when it wraps at ≤~820px
  center width) + bottom dock (`.app-region--bottom`, App.css:107) — content-driven up to a fixed
  **320px** cap (44% of a 720px viewport), no collapse affordance. Measured at 1280×720 with
  Results open: React Flow surface **800×202px**; with the first-run journey checklist visible
  (146px — the §13 tester arrival state): **800×40px**.
- **RC-2 — Fixed-size overlays in a shrinking pane.** MiniMap 200×150 (RF default), Legend up to
  320×81 (top-right), Controls 26×104 (bottom-left), M14.3 readout bottom-center with
  `max-width: 34rem` (544px). In a 202px pane: legend+minimap overlap each other (81+150 > 202);
  a pinned readout (measured 370×107) touches the minimap's left edge exactly and overlaps the
  legend by 16px; a max-width readout would overlap the minimap by ~90px at 1280 / ~44px at 1366 /
  ~27px at 1440. (This is FD-8's deferred item L-3 — see Task 0.) Scope note: chrome sitting over
  *graph content* (e.g. Controls over an edge) is inherent to a pannable infinite canvas and is
  out of scope; only chrome↔chrome collision is in scope.
- **RC-3 — Inspector content wider than its panel.** Right region fixed 260px (244 client); with a
  node selected at an evaluated session, content measures **319px** → internal horizontal
  scrollbar. Culprit: the At-session facts reuse TraceView's row CSS (`.trace-event__row`,
  min-content 300px; `.trace-event__cell--key { min-width: 6rem }`, App.css:2183-2190), sized for
  the wide dock, not a 260px sidebar.
- **RC-4 — Zero responsive machinery.** No `@media` and no `100vw` anywhere in `web/src`.
- **RC-5 (latent) — Strategy bar has no shrink protections.** Fits today at 1280 (measured
  `scrollWidth` == viewport even with the dataset chip populated) but `.sbar__name`/`.sbar__chip`
  have no `min-width: 0`/ellipsis — a long strategy name would force document horizontal scroll.

**Already correct (bounds the pass):** document-level overflow discipline holds everywhere
measured — `scrollWidth == clientWidth` at all four viewports, both themes, component view.
`box-sizing`, `min-height: 0`, and internal panel scrolling are in place. This pass is about the
vertical budget and overlay collisions, not document overflow.

## 3. Product decisions

- **D1 Inspector: fixed 260px; fix content, not frame.** Scoped wrap for At-session trace rows +
  `overflow-wrap: anywhere` on long mono identifiers. No collapse, no drawer, no new state.
- **D2 Dock: fixed open height `clamp(160px, 30vh, 320px)` + collapse toggle.** Stable across
  tabs (no height jump on tab switch); 216px @720 / 230 @768 / 270 @900 / 320 @1080. One chevron
  in the tab strip collapses to the tab row (~31px); one boolean in App state; internal scrolling
  preserved. Biggest reclaim: RF at 720 goes 202 → ~336 open / ~490 collapsed.
- **D3 Canvas minimums (asserted in tests):** RF pane ≥ 300px tall with dock open, ≥ 450px
  collapsed, ≥ 700px wide with both side panels, at every target viewport (journey dismissed).
  Stage strip goes single-row (`nowrap` + internal `overflow-x: auto`). The extraction toolbar
  stays as-is (folding its idle button into the strategy bar would move an M13 entry point right
  before §13; recorded as optional later cleanup).
- **D4 Readout/minimap: avoid collision by sizing, not moving.** Readout `max-width: 22rem` below
  1500px viewport width; minimap ~148×111 below 1500px width **or** 830px height (the OR covers
  1440×900). Positions unchanged (Legend top-right, MiniMap bottom-right, Controls bottom-left,
  readout bottom-center). No readout semantics change.
- **D5 Top bar: compress, never wrap, never remove.** Ellipsis-shrink on `.sbar__name` and the
  dataset chip; Validate/Run/Save and the cursor stay rigid. Journey checklist: steps render as an
  inline wrapped row at `max-height: 820px` (146px → ~64px).
- **D6 Journey-visible canvas minimum — approved ≥260px (review recommendation, 2026-07-24),**
  for the journey-visible transient state only, **conditional on the harness also proving**: (a)
  an edge remains hover/pin-reachable in that state, and (b) dismissing the checklist restores the
  ≥300px floor and collapsing the dock restores ≥450px. Computed basis: journey compact (~64px) +
  Results open at 1280×720 → RF ≈ 272px. Fallback levers if the assertion cannot hold (founder
  decisions, not Opus's): 26vh dock, further journey compaction, or journey auto-dismiss on first
  run.

```js
// Owns the scratch DB lifecycle around one `playwright test` invocation. The config cannot do
// this itself: it is evaluated multiple times and has no TestInfo, so it only READS the env var.
import { mkdtempSync, rmSync } from 'node:fs'
import { tmpdir } from 'node:os'
import { join } from 'node:path'
import { spawnSync } from 'node:child_process'

const dir = mkdtempSync(join(tmpdir(), 'quantize-e2e-'))
const result = spawnSync('npx', ['playwright', 'test', ...process.argv.slice(2)], {
  stdio: 'inherit',
  shell: process.platform === 'win32',
  env: { ...process.env, QUANTIZE_E2E_DB: join(dir, 'e2e.db') },
})
rmSync(dir, { recursive: true, force: true })
process.exit(result.status ?? 1)
```

**1c — `web/playwright.config.ts`.** Fail loud if launched without the runner:

