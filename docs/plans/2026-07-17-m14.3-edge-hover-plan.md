**Goal:** hovering or keyboard-pinning a graph edge (and a node card's output-port row) while a run is selected and the cursor sits on an evaluated session shows the **served value summary flowing across that connection at that instant**, as pure presentation over the existing `GET /v1/runs/{id}/values` endpoint.

**Architecture:** zero backend change. Every projected edge carries its tap address (`source` + `sourceHandle`, `web/src/document/flow.ts:360‚Äì367`). A fixed readout panel owns a dwell-gated, uncached fetch whose result is stored under an **activation generation** (tracked by state-backed render-phase transition) plus the **full request-identity tag**, and rendered only while both match ‚Äî the display is a pure function of current props in every commit, including the commit in which the activation changes. Pinning is two explicit user-intent gestures (mouse `onEdgeClick`; keyboard Enter/Space via a capture-phase handler reading the focused edge's `data-id`) ‚Äî RF selection is never consulted. Hover/pin state carries its originating trail key and render-gates on it.

**Tech stack:** React 18 + `@xyflow/react` 12.8.6 (`web/package.json:17`), vitest + testing-library, existing `getNodeValue` (`web/src/api/client.ts:224`). No new dependencies.

7. **Result lifetime ‚Äî the full rule, mechanism included (audit rounds 2‚Äì4):**
   - **Activation key:** `activationKey = address === null ? '‚àÖ' : tag`, where `tag = ` `` `${runId}|${cursor}|${componentPath.join('/')}|${nodeId}|${outputPort}` ``. Every key CHANGE ‚Äî including `A ‚Üí ‚àÖ ‚Üí A` (leave + re-hover of the same address) ‚Äî is a new activation.
   - **Generation mechanism (round-4 finding 1):** a **state-backed render-phase tracker**, the `Canvas.tsx:442` `prevViewKey` pattern: `const [tracker, setTracker] = useState({ key: activationKey, gen: 0 })`; **during render**, `if (tracker.key !== activationKey) setTracker({ key: activationKey, gen: tracker.gen + 1 })`. React re-renders before committing, so in the very commit where the activation changes, `currentGen` has already advanced and a stored result with the old `gen` **cannot render even once**. State, never a render-mutated ref (the M13.8 round-7 StrictMode lesson: render-phase resets pair only with state); never an effect (post-commit = one stale frame ‚Äî the mechanism this plan rejects for trail gating must not be smuggled back in here).
   - **Requests:** the dwell timer lives in one effect keyed on `[tracker.gen]` (plus probe/address via the key); it captures `tracker.gen` at launch; cleanup cancels the timer. A resolution is stored only if its captured gen is still current, and the store is `{gen, tag, result}`.
   - **Render rule:** show the stored result only when `stored.gen === tracker.gen && stored.tag === currentTag`.
   - Pinned consequences (tested): re-hover of the same address never shows the old value during the new dwell (same tag, old gen ‚Äî cross-activation reuse would be the ¬ß10.5 cache smell); a pending old-activation response can never overwrite a newer one; a run/cursor switch under a pinned address blanks the display in the switch commit itself.
8. **Pinning does NOT use RF selection.** Verified chain: validation highlighting seeds `selected: true` (`Canvas.tsx:535`); RF's `getSelectionChanges` emits nothing on Enter over an already-selected edge (`dist/esm/index.mjs:778`). Pin paths:
   - **Mouse:** `onEdgeClick(_, edge) ‚Üí pin(addressOf(edge))`.
   - **Keyboard (round-4 finding 2):** a probe-gated **capture-phase** `onKeyDownCapture` handler on the canvas wrapper div. Accepted keys: `'Enter'` and `' '` (the browser's actual Space `key` value, matching RF's `elementSelectionKeys`). If the event target is inside an RF edge element ‚Äî `closest('.react-flow__edge')?.getAttribute('data-id')`, resolved against `rfEdges` ‚Äî then **`preventDefault()` AND `stopPropagation()`**, and pin. `stopPropagation` is load-bearing: RF's pan-activation listener sits on **`window`** (`dist/esm/index.mjs:1971`) and still receives a bubbling event whose default was prevented ‚Äî without stopping propagation, Space-to-pin would also activate canvas panning. The capture-phase element handler runs before the window-level bubble listener, so stopping there is sufficient. **On a miss (no edge under focus), the handler does nothing at all** ‚Äî no pin, no preventDefault, no stopPropagation ‚Äî RF's own Space/pan behavior (which intentionally prevents Space's default) proceeds untouched; do NOT assert "default not prevented" outside edges, that would assert against RF itself.
   - RF v12 renders focusable edges as `.react-flow__edge` groups carrying `data-id` + `tabindex` ‚Äî **Task 4's first keyboard test asserts the `data-id` attribute on the rendered DOM**.
   - Read-only view (D-39): edges need only per-element `focusable: true` when a probe exists; `elementsSelectable={false}` untouched.
9. **Trail-change gating is render-phase:** `viewKey` = `component:${componentId}@${version}` (`Canvas.tsx:438`) ‚Äî unchanged across a same-definition **instance** switch (`App.tsx:559` replaces the trail directly). Hover/pin state stores its originating `trailKey = (componentTrail ?? []).map(e => e.instanceId).join('/')`; the **effective address is a pure derivation** ‚Äî `effectiveAddress(state, currentTrailKey)` ‚Üí address on key match, else null ‚Äî exported and unit-tested. 11. **Canvas structure:** 993 lines; `viewKey` remounts the inner `<ReactFlow>` (state on the outer Canvas); `CanvasChromeContext` feeds custom node cards (Task-6 channel); RF `Panel` precedent: Legend `top-right` (`Canvas.tsx:974`).
